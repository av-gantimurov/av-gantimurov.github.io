Представляет собой ВПО семейства Мамонт для ОС Android и предназначен для скрытного получения, отправки SMS.

Реквизиты С2 хранятся в нативной библиотеке в зашифрованном виде.

В коде реализована возможность выявления факта функционирования в контролируемом окружении (песочнице), при этом который никак не используется.

## Алгоритм работы

За основу взято приложение [Simple Mobile Tools SMS](https://github.com/SimpleMobileTools/Simple-SMS-Messenger)

В коде выявлены следующие `Activity`, содержащие вредоносный функционал:

- `SplashActivity`
- `ActivationActivity`

Кроме этого, приложение регистрирует обработчик входящих SMS (`Receiver`) для их перенаправления на удаленный сервер.

Вызывается `SplashActivity` при старте приложения, в дальнейшем
вызывает `ActivationActivity`, `MainActivity`

При вызове `ActivationActivity`:

- Запрашивает необходимые разрешения (SMS `android.app.role.SMS`).
- Запрашивает номер телефона. Отправляет на С2 через `/register.php` информацию об идентификаторе устройства (`android_id`) и номере телефона.
- Запрашивает установить приложением для обработки SMS.

В случае успешной регистрации устройства инициализируется основной функционал (`scheduleInitialWorkers`).

![Код активации](image-2.png)

`WorkerScheduler` запускает обработку SMS команд, отправку информации об установленных приложениях и информации об устройстве.

```java
com.simplemobiletools.smsmessenger.workers.AppListWorker;
com.simplemobiletools.smsmessenger.workers.DeviceInfoWorker;
com.simplemobiletools.smsmessenger.workers.SmsCommandWorker;
```

![Вызов кода обработчиков](image-3.png)

`SmsCommandWorker` периодично запрашивает команды на отправку SMS с удаленного сервера и отсылает информацию о работе устройства (`heartBeat`)

Формат полученной команды

```json
{
    "command_id": "",
    "recipient": "",
    "text": "",
}
```

SMS отправляется с указанными параметрами через Android API `smsManager.sendMultipartTextMessage`.

![Код отправки SMS ](image-4.png)

В противном случае просто отсылается `heartbeat` через Retrofit с идентификатором устройства.

![Код отправки информации об устройстве](image-5.png)

В случае критической ошибки при отправке SMS отправляет сообщение об ошибке через `/update_status.php`

![Код отправки сообщения об ошибке](image-7.png)

`AppListWorker` через `PackageManager.getInstalledApplications` осуществляет сбор названий всех приложений.
Полученный список отправляется в адрес C2 через Retrofit `/update_apps.php`.

`DeviceInfoWorker` собирает информацию об устройстве.

![Код сбора информации об устройстве](image-6.png)

Собирает `Build.MANUFACTURER`, `Build.MODEL`, размер экрана, MAC адрес, User-Agent (`http.agent`)

Отправляется в формате JSON через `update_device_info.php`

```json
{
    "deviceId": "",
    "deviceModel": "",
    "macAddress": "",
    "screenSize": "",
    "userAgent": "",
}
```

## Перенаправление SMS

В ходе работы регистрирует и запускает обработчик входящих SMS сообщений

```xml
<receiver
    android:name="com.simplemobiletools.smsmessenger.receivers.SmsReceiver"
    android:permission="android.permission.BROADCAST_SMS"
    android:exported="true">
    <intent-filter>
        <action android:name="android.provider.Telephony.SMS_DELIVER"/>
    </intent-filter>
</receiver>
```

Все входящие SMS перенаправляются на удалённый С2 сервер через `/incoming_sms.php`.

![Код перенаправления SMS](image-8.png)

## Антиотладка

Приложение содержит класс `AntiReverse` в `com.simplemobiletools.util`.

В данном классе реализован метод `checkAndKill`, осуществляющий проверку на наличие эмулятора `isEmulator`, отладки `isDebugging`, модификации пакета `isTampered`, использования frida `isFridaPresent` и модулей Magisk `isMagiskPresent`.

![Код антиотладки](image.png)

Если выявлен факт отладки или запуска в контролируемом окружении, то приложение завершается с отправкой сообщения "ВСКРЫТИЕ: {Build.BRAND} {Build.MODEL}" в телеграм чат.

![Код отправки сообщения](image-1.png)

**Хоть данный функционал и реализован, но никак в коде программы не вызывается.**

## Сетевое взаимодействие

URL для отправки данных зашит в нативной библиотеке и извлекается через метод `Secret.INSTANCE.getBaseUrl()`.

```java
Retrofit.Builder baseUrl = new Retrofit.Builder().baseUrl(Secret.INSTANCE.getBaseUrl());
```

```java
public final class Secret {
    public static final Secret INSTANCE = new Secret();

    public final native String getBaseUrl();

    private Secret() {
    }

    static {
        System.loadLibrary("native-lib");
    }
}
```

URL хранится в Base64 в зашифрованном виде в глобальной переменной.
Для шифрования используется XOR с ключом, который также хранится в глобальной переменной.

![Код расшифровки URL](image-9.png)

В всех нативных библиотеках URL и ключ для расшифровки одинаковый.

Для взаимодействия c C2 используется библиотека Retrofit, данные передаются в формате JSON.

```java
Retrofit.Builder baseUrl = new Retrofit.Builder().baseUrl(Secret.INSTANCE.getBaseUrl());
```

```java
public interface ApiService {
    @POST("/incoming_sms.php")

}
```
